## Scope and Methodology Notes

**Channel Classification:** PETRA evaluates payment channels against the requirement that all original payments must have been transmitted via encrypted channels. The following determinations apply to channel types observed in this dataset:

| Channel | Encryption Status |
|---|---|
| Wire Transfer | ✅ Encrypted — bank-to-bank transmission over secured networks |
| ACH | ✅ Encrypted — NACHA-governed secure electronic network |
| ACH - CBP/ACE | ✅ Encrypted — government-mandated secure ACE portal |
| EFT | ✅ Encrypted — electronic funds transfer over secured channels |
| Corporate Card | ✅ Encrypted — PCI-DSS governed encrypted card network |
| Single Entry Bond | ✅ Encrypted — CBP-regulated bond instrument, secured channel |
| Monthly Statement | ✅ Encrypted — consolidated CBP billing through ACE system |
| Check | ⚠️ SECURITY FLAG — physical instrument; not an encrypted electronic channel |
| [BLANK] | ⚠️ SECURITY FLAG — channel unknown; compliance cannot be determined; assumed non-compliant per PETRA policy |

**Amount Verification Methodology:** PETRA independently verifies: (1) Cost before Tariff = Quantity × Unit Cost; (2) Tariff Amt Paid = Cost before Tariff × Tariff Rate; (3) Total Payment = Cost before Tariff + Tariff Amt Paid; (4) Refund Claimed = Tariff Amt Paid. Any variance, including rounding, is flagged.

**Routing Verification:** PETRA is not in possession of approved vendor banking records. Accordingly, all refund routing is flagged as PENDING VERIFICATION — requiring cross-check against approved vendor banking records before any refund can advance. This is applied uniformly to all 41 invoices.

**PETRA does not process, initiate, or recommend initiation of any refund payment.**
