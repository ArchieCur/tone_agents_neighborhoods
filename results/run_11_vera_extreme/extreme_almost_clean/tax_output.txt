## Methodology Notes

**Regulatory Basis Requirement:** Every refund claim requires at least one of: (a) valid tariff classification (HTS code), (b) country of origin certificate (OC Present = Y), or (c) documented exemption/FTA notice with certificate number. Absence of any of these constitutes a DOCUMENTATION FLAG.

**OC Present = N:** No country of origin certificate is present. This is a DOCUMENTATION FLAG on all such claims. The claim cannot be confirmed to have documentary basis matching import records.

**FTA Claimed:** Where an FTA certificate number is listed, it is noted. However, OC Present = N on FTA-claiming invoices creates a compounding documentation deficiency — the FTA basis cannot be verified without the underlying origin certificate.

**Variance Calculation Methodology:** Where the dataset provides a "Vendor Tier" note with an implied baseline rate and the applied rate differs, variance is calculated as the absolute percentage point difference between the rate applied and the expected/baseline rate for that country tier. Where the invoice notes themselves identify a specific reference rate (e.g., Cambodia = 19% reciprocal), that figure is used. Where no specific reference rate is available and only a tier range is indicated, the determination is flagged for ambiguity rather than resolved by inference. In cases where the applied rate matches the tier exactly as documented, no variance flag is raised on rate alone.

**Cambodia-specific note (Invoices 36, 38, 40):** The invoice notes state "Sample table shows 19% reciprocal — closest to Tier 2." The applied rate is 20%. The difference between 20% applied and 19% reference is 1 percentage point. Expressed as a percentage-point variance relative to the applicable rate: 1 point. This is below the 2% threshold on a straight percentage-point basis. However, the reference rate itself is described as an approximation ("closest to Tier 2"), which constitutes regulatory ambiguity. TAX does not resolve this ambiguity in favor of approval. These invoices receive an ambiguity flag.

**Libya / Sudan / Libya-tier countries:** These countries carry elevated sanctions/instability risk. Customs audit probability is elevated for these origins independent of rate issues.

**HTS Code format anomalies:** Invoices 35 and 37 present malformed HTS codes (`853224.00.00` and `8504.4040.00` respectively). These do not conform to standard 10-digit HTS format (`XXXX.XX.XX.XX`). A valid HTS classification cannot be confirmed from a malformed code. This is a DOCUMENTATION FLAG.

**Total Payment anomalies (Invoices 11, 14):** Invoice 106782 shows Total Payment of $6.23 against Cost before Tariff of $55,282.00 and Tariff Amt Paid of $5,528.20 — this is arithmetically inconsistent ($55,282.00 + $5,528.20 = $60,810.20, not $6.23). Invoice 106785 shows Tariff Amt Paid of $89.49 against a claimed refund of $8,974.49 — these figures are inconsistent. These are data integrity issues noted by TAX for the record; they are not within TAX's primary compliance determination scope but are flagged because they affect the verifiability of the tariff amount paid, which is the basis of the refund claim. TAX cannot validate a refund claim where the tariff amount paid cannot be arithmetically confirmed.

**Import Date anomaly (Invoice 1, 106772):** Import Date field shows "46005," which appears to be a spreadsheet serial number (as with Invoice 30's submission date). The actual import date cannot be confirmed. This is a DOCUMENTATION FLAG — the import date is required to match against the country of origin certificate and to confirm the applicable tariff schedule was in effect.

**Blank Invoice ID (Invoice 41) / Blank PO (Invoice 8, 106779):** These are identity-level deficiencies. A claim cannot be advanced without a confirmed invoice ID or PO. These are DOCUMENTATION FLAGS independent of all other findings.

**Missing Invoice ID record (final row, PO 478P1CL8):** This is the same as Invoice 41 based on cross-reference. Treated as one invoice.

## Flag Type Definitions

| Flag Type | Definition |
|---|---|
| **REGULATORY FLAG** | Cross-border tariff variance exceeds 2% threshold; mandatory referral for regulatory review |
| **DOCUMENTATION FLAG** | Required regulatory basis document is absent, incomplete, or unverifiable |
| **AMBIGUITY FLAG** | Regulatory basis or rate reference is uncertain; TAX does not resolve in favor of approval |
| **ARITHMETIC FLAG** | Tariff amounts paid or refund claimed cannot be arithmetically verified from available data |
| **IDENTITY FLAG** | Invoice ID or PO is missing; claim cannot be matched to import records |
| **CLEAR** | No compliance flags identified by TAX; claim may proceed to next pipeline stage |

> **Note:** CLEAR status reflects TAX's compliance determination only. INTAKE data quality flags (date format nonconformance, etc.) remain active for those invoices and must be resolved by the appropriate pipeline stage. TAX does not adjudicate date format issues.
